Doublons Critiques

🚨 Doublons Critiques Identifiés

1

Services VisualAI Dupliqués CRITIQUE

Problème: Deux implémentations différentes de VisualAI existent:

server/services/visualAI.ts (1202 lignes) - Complet avec fraude detection, modération, reporting
server/services/visualAIService.ts (307 lignes) - Simplifié pour monitoring Live Shows

Impact: Confusion dans l'architecture, risque d'utiliser le mauvais service, maintenance difficile

2

Services Bunny Fragmentés CRITIQUE

Problème: Trois services Bunny avec fonctionnalités qui se chevauchent:

server/services/bunnyClient.ts (65 lignes) - Opérations API basiques
server/services/bunnyVideoService.ts (341 lignes) - Gestion vidéo complète + tokens
server/services/bunnySignedUrl.ts (151 lignes) - Génération URLs signées CDN

Impact: Redondance dans génération de tokens, confusion sur quel service utiliser

3

Dossier attached_assets Non Nettoyé IMPORTANT

Problème: 110 fichiers de documentation/specs obsolètes ou dupliqués (373 KB)

Formules et Repartition_Categorie_CORRECTIF_160925 (3 versions)
Investment_Rules_160925 (2 versions)
Complements_Manquants 160925 (2 versions)
+ 103 autres fichiers de specs/docs

Impact: Confusion sur quelle version est actuelle, espace disque gaspillé, repo surchargé

4

Routes Admin Redondantes IMPORTANT

Problème: Logique admin fragmentée dans plusieurs fichiers:

server/services/adminRoutes.ts
server/services/adminConsole.ts
server/services/adminAccessService.ts
server/routes/adminDashboardRoutes.ts

Impact: Code difficile à maintenir, risque de régression lors des modifications

5

Sitemaps Dupliqués MINEUR

Problème: Sitemaps présents dans 2 locations:

client/public/sitemaps/
scripts/client/public/sitemaps/

Impact: Risque de désynchronisation, confusion sur la source de vérité

✅ Recommandations d'Optimisation

1

Fusionner les Services VisualAI PRIORITÉ 1

Action:

  • Garder visualAI.ts comme service principal (plus complet)
  • Intégrer la fonctionnalité de monitoring Live Show depuis visualAIService.ts
  • Supprimer visualAIService.ts après migration
  • Mettre à jour tous les imports

Gain: -307 lignes, architecture clarifiée, maintenance simplifiée

2

Consolider Services Bunny PRIORITÉ 1

Action:

  • Garder bunnyVideoService.ts comme service principal
  • Intégrer les fonctions CDN signed URLs depuis bunnySignedUrl.ts
  • Déprécier bunnyClient.ts (API de bas niveau)
  • Créer un service unifié bunnyService.ts

Gain: API unifiée, élimination de la duplication de logique de tokens

3

Nettoyer attached_assets PRIORITÉ 2

Action:

  • Créer dossier docs/ pour documentation actuelle
  • Archiver ou supprimer fichiers obsolètes avec timestamps
  • Garder seulement la version la plus récente de chaque spec
  • Créer un index README dans docs/

Gain: ~100 fichiers supprimés, documentation claire et accessible

4

Unifier Routes Admin PRIORITÉ 2

Action:

  • Créer structure modulaire dans server/admin/
  • Séparer: routes, services, middleware
  • Centraliser la logique d'authentification admin

Gain: Code admin maintenable et évolutif

5

Éliminer Sitemaps Dupliqués PRIORITÉ 3

Action:

  • Garder seulement client/public/sitemaps/
  • Supprimer scripts/client/public/sitemaps/
  • Vérifier script de génération de sitemaps

Gain: Source de vérité unique pour le SEO

🎯 Plan d'Action Recommandé

Phase 1 - Services Backend (Priorité Haute)

  1. Créer server/services/bunnyService.ts unifié
  2. Migrer toutes les fonctionnalités Bunny vers service unifié
  3. Intégrer monitoring Live Show dans visualAI.ts
  4. Mettre à jour tous les imports dans le projet
  5. Supprimer fichiers obsolètes: bunnyClient.ts, visualAIService.ts

Phase 2 - Documentation et Assets (Priorité Moyenne)

  1. Créer structure docs/ propre
  2. Migrer specs actuelles vers docs/
  3. Supprimer fichiers obsolètes dans attached_assets/
  4. Créer index de documentation

Phase 3 - Structure Admin (Priorité Normale)

  1. Créer server/admin/ avec structure modulaire
  2. Consolider routes et services admin
  3. Nettoyer sitemaps dupliqués
  4. Tester compilation et fonctionnalités

⚠️ Précautions Importantes

  • Backup: Créer une branche Git avant toute modification
  • Tests: Exécuter tests après chaque consolidation
  • Imports: Utiliser recherche globale pour mettre à jour tous les imports
  • Fonctionnalités: Vérifier que chaque fonctionnalité est bien migrée
  • Build: Exécuter npm run build après chaque phase
